# Contributing to APartner

Thanks for your interest in contributing.

## Ground rules

- Be respectful and constructive.
- Follow the [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md).
- Keep changes focused and well-scoped.

## Development setup

For app setup instructions, use:

- [app/README.md](./app/README.md)

## Pull request expectations

Please include:

- A clear summary of what changed and why.
- Any relevant screenshots or notes for UI changes.
- Testing notes (what you ran and outcomes).

Before submitting, run from `app/`:

```bash
npm run verify
```

## Issue reports

When reporting a bug, include:

- Steps to reproduce.
- Expected vs actual behavior.
- Environment details (OS, Node version, browser if relevant).
